# Scripts

Purpose: validation and orchestration adapters.

Use scripts when a check should be repeatable locally. Scripts are one possible adapter; MCP tools, IDE integrations, CI checks, browser tools, and manual review can also satisfy a validation interface.

- workspace health
- project registry validation
- link checks
- naming checks
- schema checks
- preview server checks
- screenshot or visual checks

Scripts should be documented by purpose, command, inputs, outputs, and safety boundaries.
