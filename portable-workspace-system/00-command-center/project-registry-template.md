# Project Registry Template

Use this shape to make projects observable to humans and agents.

| Field | Meaning |
|---|---|
| ID | Stable kebab-case project id. |
| Name | Human-readable project name. |
| Area | Workspace area, such as active, incubation, brand, tools, archive. |
| Path | Workspace-relative folder path. |
| Status | Draft, active, paused, archived, shipped, or blocked. |
| Owner | Human/team owner or `unassigned`. |
| Summary | One-line purpose. |
| Run/preview | Command, URL, static entry, or `n/a`. |
| Docs | README, architecture, status, or handoff link. |
| Last reviewed | Absolute date. |
