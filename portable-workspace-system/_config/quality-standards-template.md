# Quality Standards Template

## General Quality Bar

A good output should be accurate, specific, useful, easy to revise, grounded in provided inputs, and explicit about assumptions and missing information.

## Review Checklist

- Did I read the right routing docs?
- Did I use only required or relevant inputs?
- Did I follow local folder rules?
- Did I update source-of-truth docs when state changed?
- Did I keep generated output separate from canonical docs?
- Did I run or name validation?
- Did I avoid exposing secrets?
